# PRD (Product Requirements Document)

## 1. 프로젝트 개요
- **서비스명**: AIARA Space
- **목적**: 비즈니스 사용자들이 AI를 더 효과적으로 사용할 수 있도록 검증된 프롬프트를 제공하고, 나쁜 프롬프트를 좋은 프롬프트로 교정해주는 도구 제공.
- **주요 타겟**: 기획자, 마케터, HR 담당자, 개발자, 경영진 등 사무직 종사자.

## 2. 사용자 페르소나
1. **김기획 (기획자)**: 보고서 구조를 잡는 데 어려움을 겪으며, AI에게 어떻게 질문해야 할지 모름.
2. **이마케 (마케터)**: 매일 SNS 카피를 써야 하며, 창의적인 영감이 필요함.
3. **박인사 (HR)**: 채용 공고나 사내 공지문을 정중하고 전문적으로 작성하고 싶음.

## 3. 핵심 기능 요구사항 (FR)
- **FR-1**: 10개 카테고리로 분류된 총 100개의 비즈니스 프롬프트 노출.
- **FR-2**: 카테고리 필터링 및 검색 기능을 통한 프롬프트 탐색.
- **FR-3**: 각 프롬프트는 원클릭으로 클립보드 복사 가능해야 함.
- **FR-4**: 한국어/영어 언어 전환 기능 제공.
- **FR-5**: 사용자가 입력한 짧은 문장을 '페르소나+맥락+제약조건'이 포함된 고급 프롬프트로 변환해주는 AI Optimizer 제공.

## 4. 비기능 요구사항 (NFR)
- **NFR-1**: 모던한 다크 테마 디자인 (Z-Design 스타일).
- **NFR-2**: 반응형 레이아웃 (데스크톱 우선).
- **NFR-3**: 프롬프트 최적화 응답 속도 (3초 이내 권장).
- **NFR-4**: GitHub Actions (Node.js 24) 최신 환경 준수.

## 5. 로드맵 (Future Roadmap)
- [ ] 사용자 커스텀 프롬프트 저장 기능.
- [ ] 프롬프트 활용 결과물 피드백 시스템.
- [ ] 더 많은 AI 모델 선택 기능 (Claude, GPT-4 등).
- [ ] 카테고리별 활용 가이드 영상 링크 연동.
